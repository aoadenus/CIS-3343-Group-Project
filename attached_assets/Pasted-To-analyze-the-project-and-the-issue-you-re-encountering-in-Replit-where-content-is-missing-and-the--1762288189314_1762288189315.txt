Common Causes and Diagnostics
Port and .replit Configuration

If the port used by your app internally (e.g., 3000, 5000, or another) does not match the port exposed via externalPort = 80 in your .replit config, the app may appear to load some static resources but fail to mount React, resulting in blank/missing content and unresponsive navigation. Make sure your run command, internal port, and externalPort match what's required by Replit Autoscale deployment.‚Äã‚Äã

Vite Proxy and Path Issues

If you are using Vite's proxy feature and endpoint requests are not routed properly due to path, target, or CORS config issues, this can lead to fetch errors, blank screens, or incomplete renders. Double-check your vite.config.js to ensure proxy paths use relative URLs (/api/... rather than hardcoded domains), and that backend responses have permissive CORS headers.‚Äã

Heavy Components or UseEffect Delays

Rendering-heavy or animation-heavy components in React can cause the browser to hang, especially if animations are ongoing during route transitions. If a useEffect hook or a data-fetch blocks the main thread, any navigation during an animation can trigger a freeze.‚Äã

Janeway/Preview Bug

There have been multiple reports where Replit's Janeway proxy breaks routing or fails to serve compiled assets properly for React apps, resulting in blank previews or frozen UI‚Äîwhile the app works fine outside Replit. Sometimes, the problem only appears on the public/external URL, not on the internal preview.‚Äã
